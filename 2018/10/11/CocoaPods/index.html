如何把自己的开源代码发布到CocoaPods上

2018-10-11   CocoaPods 

由于所在公司开发团队同时维护了三个APP，所以会经常出现三个APP中同时需要相同的诸如工具类，框架，业务等代码，如果每个APP都各自写一遍，未免太浪费时间，所以会经常需要写一些通用的代码放在CocoaPods上。今天就来记录一下如何把自己的代码放给CocoaPods上供很多很多很多人都能直接用嘻嘻嘻

在这篇文章中，我忽略了CocoaPods的安装，原理等 如果有这方面需求可参考https://www.jianshu.com/p/83baebf049b8

先梳理大致过程

  • 把自己的代码开源到GitHub上

  • Trunk相关操作

  • 新建 .podspec文件并提交

  • 更新版本

把自己的代码开源到GitHub上

登录GitHub 新建仓库

填写（仓库名字（Repository name）、仓库权限（Public）、readme、license）

本地创建文件夹

好的，现在GitHub上已经开源了想要分享的代码

Trunk相关操作

查看cocoaPods版本

pod --version

确定版本大于0.33，如果小于这个版本 升级cocapods

sudo gem install cocoapods

注册Trunk账号

pod trunk register 邮箱地址 ‘用户名’

登录邮箱，打开邮箱里给到的链接，会看到注册成功

查看注册信息

pod trunk me

新建 .podspec文件并提交

什么是.podspec文件

.podspec文件是为了让CocoaPods搜索引擎知道该分享代码的作者、版本号、概要、描述、源代码地址、部署版本等描述信息。 .podspec文件可以自动生成，这里建议在github上找一个.podspec文件修改一下参数即可。毕竟很多开源的框架写的.podspec肯定是能够满足我们所有的信息描述又不至于产生太多的冗余。

新建 .podspec文件

pod spec create xxx //生成xxx.podspec文件

新建结束后 打开.podspec 参考知名开源框架的写法写自己的信息，这一步很容易出错，务必仔细认真

说明几个参数：

1、s.license

Pods依赖库使用的license类型，在创建GitHub仓库的时候创建的license类型写上即可
2、s.source_files

表示源文件的路径，注意这个路径是相对podspec文件而言的。
3、s.frameworks

需要用到的frameworks

4、s.name

名称，pod search 搜索的关键词,注意这里一定要和.podspec的名称一样,否则报错

5、s.version

版本号

6、s.ios.deployment_target

支持的pod最低版本

7、 s.summary:

简介

8、s.homepage

项目主页地址

9、 s.license:许可证

source_files:写法及含义建议大家写第一种或者第二种

“*” 表示匹配所有文件
“*.{h,m}” 表示匹配所有以.h和.m结尾的文件
“**” 表示匹配所有子目录

验证.podspec文件

pod spec lint

验证失败后 要仔细查看下.podspec文件

提交.podspec文件到trunk中

pod trunk push xxx.podspec

好了，到这里就可以使用刚刚提交给pods的代码了（很多人出现搜索不到的问题，只能pod setup一下，但我没有出现该问题）

更新版本

如果在版本使用期间发现了bug或者有新的东西提交，那么就需要把提交流程重新走一遍

  • 提交代码
  • 打tag
  • 修改.podspec文件中的version字段
  • 提交.podspec文件到trunk

以上即可

碎碎念

tag相关操作：

git tag // 查看tag
git tag -d 0.0.1 //删除一个指定的tag
git push origin :refs/tags/1.0.0 // push到远程tag
